## 三、測試準備

1. **登入 UAT**：開啟 `https://cmp-uat-100.metaage.com.tw`，以測試帳號完成 Microsoft 帳號登入。
2. **確認部署**：確認本次三檔修正已部署至 UAT（可於新增訂單觸發錯誤時觀察通知內容是否已非 `[object Object]`）。
3. **安裝 XHR 攔截器／回應改寫器**（詳見附錄 A、B）。頁面導航／重載會清空攔截器，需重新安裝。
4. **建立測試訂單**：依 CLAUDE.md「訂單建立通用注意事項」建立一張可送出的訂單（建議 AWS 用量制品項），用於 A 群組；訂變單測試需先有一張已核准訂單以發起訂變單（B 群組）。
5. **附件上傳**：A 群組需於新增訂單頁上傳附件至**超過數量上限（>10 個）**以觸發後端報錯。
6. **截圖路徑**：`documents/CMP-4501/screenshots/`，命名 `{TC編號}-{描述}.png`。

## 七、附錄

### 附錄 A — XHR 攔截器（觀察 order / modify / material 請求）

```js
agent-browser eval "
(function() {
  window.__cap = [];
  const oOpen = XMLHttpRequest.prototype.open;
  const oSend = XMLHttpRequest.prototype.send;
  XMLHttpRequest.prototype.open = function(m, u) { this.__m = m; this.__u = u; return oOpen.apply(this, arguments); };
  XMLHttpRequest.prototype.send = function(b) {
    const x = this, u = x.__u || '';
    if (/\/(order|order\/modify\/)/.test(u)) {
      const it = { m: x.__m, u: u, body: b };
      window.__cap.push(it);
      x.addEventListener('load', () => { it.s = x.status; it.r = x.responseText?.slice(0, 500); });
    }
    return oSend.apply(this, arguments);
  };
  return 'ok';
})();
"
```

> 注意：`postOrderWithFullError` / `createModifyOrderWithFullError` 皆走 Angular `HttpClient`（內部使用 XHR），可被攔截。

### 附錄 B — 回應改寫器（模擬各類錯誤型別）

用於 TC-03／04／07／08：攔截目標請求並回傳自訂錯誤狀態與 body，以驗證前端各錯誤分支。

| 測項 | 模擬回應 body（`error.error`）| 目的 |
|------|------|------|
| TC-03／TC-08 | `{ info: { code: 'bpm_fail_xxx', message: '流程錯誤訊息' } }` | 走 bpm_fail 分支 |
| TC-04／TC-07 | `{ message: '未知錯誤' }`（無 `info.code`）| 走 `new Error()` 分支，驗證 `err.code` 防護 |

> 實作可用 Service Worker 或覆寫 `XMLHttpRequest`/`fetch` 回傳，依 UAT 實際可行方式擇一；若 UAT 不便偽造回應，改以後端配合或單元測試補充，並於對應 TC 標註。
