# 整合測試項目 — 微軟下單檢核改用 SSE 架構
**關聯 Jira：** CMP-4280（前端）、CMP-4266（後端）

**測試範圍說明：**
微軟訂單的「下單檢查」功能改版。原本點擊後需等候很長時間才會一次顯示結果（容易 timeout），改版後會即時顯示檢查進度（逐步回傳結果）。測試時需搭配微軟品牌的訂單 / 訂變單進行操作。

**操作路徑：** 訂單詳情頁 → 子單區塊 → 點擊「下單檢查」按鈕

---

## 一、Modal 開啟前置驗證

> 點擊「下單檢查」按鈕時，系統應先檢查必要欄位是否齊全，不齊全則不開啟檢查視窗。

- [ ] **1.1** 子單的 Cloud ID 欄位為空 → 點擊「下單檢查」後不開啟檢查視窗，畫面顯示黃色警告提示
- [ ] **1.2** 子單內沒有任何產品 → 點擊「下單檢查」後不開啟檢查視窗，畫面顯示黃色警告提示
- [ ] **1.3** 訂單的 Partner ID 為空 → 點擊「下單檢查」後不開啟檢查視窗，畫面顯示黃色警告提示
- [ ] **1.4** 產品的原廠料號格式異常 → 點擊「下單檢查」後不開啟檢查視窗，畫面顯示錯誤訊息
- [ ] **1.5** 上述欄位皆正確填寫 → 成功開啟「下單檢查」視窗，檢查期間背景畫面不可操作

---

## 二、檢查進度顯示

> 開啟檢查視窗後，系統會與後端建立即時連線，逐步回傳檢查進度。視窗上方會顯示步驟進度條，最後一階段會逐筆顯示產品驗證結果。

- [ ] **2.1** 視窗開啟後 → 上方出現步驟進度條，正確顯示後端回傳的各步驟名稱（如：驗證合作夥伴資訊、取得原廠客戶資訊…等）
- [ ] **2.2** 每個步驟開始時 → 該步驟在進度條上標示為「進行中」；完成後標示為「已完成」
- [ ] **2.3** 訂單不含需額外驗證的商品 → 進度條只顯示 2 個步驟，流程正常完成
- [ ] **2.4** 訂單包含需額外驗證的 RI 商品（如 Reserved VM Instance）→ 進度條顯示完整 4 個步驟
- [ ] **2.5** 最後一步驟（逐筆驗證產品）執行時 → 下方產品清單即時更新每筆驗證結果，不需等全部跑完才一次顯示

---

## 三、驗證結果情境

> 依據檢查結果的成功 / 失敗，畫面有不同的顯示方式。

- [ ] **3.1** 全部檢查通過 → 所有步驟標示完成，視窗底部出現「關閉」按鈕，可正常關閉視窗
- [ ] **3.2** 驗證合作夥伴失敗（第 1 步） → 進度條停在第 1 步並標示為錯誤（紅色），下方顯示錯誤原因文字，視窗無法自動關閉
- [ ] **3.3** 取得原廠客戶資訊失敗（第 2 步） → 進度條停在第 2 步並標示為錯誤，顯示錯誤原因，視窗無法自動關閉
- [ ] **3.4** 部分產品驗證失敗 → 失敗的產品顯示紅色錯誤說明文字，通過的產品顯示通過狀態；全部跑完後底部出現「關閉」按鈕
- [ ] **3.5** 全部產品驗證失敗 → 每筆產品都顯示紅色錯誤說明文字；全部跑完後底部出現「關閉」按鈕
- [ ] **3.6** 檢查途中網路斷線 → 畫面顯示連線錯誤提示，視窗不自動關閉，可手動關閉

---

## 四、驗證結果回寫至訂單

> 關閉檢查視窗後，檢查結果需正確回寫至子單的產品資料上，影響後續送單判斷。

- [ ] **4.1** 全部產品檢查通過 → 關閉視窗後，子單上的產品顯示「已檢核」狀態（如打勾圖示或標籤）
- [ ] **4.2** 部分產品檢查失敗 → 關閉視窗後，失敗的產品顯示錯誤說明，通過的產品顯示通過狀態
- [ ] **4.3** 完成檢核後修改產品資料（如數量、金額等）→ 該子單的檢核狀態還原為「未檢核」，「下單檢查」按鈕恢復為可點擊
- [ ] **4.4** 完成檢核後未修改任何資料 → 「下單檢查」顯示「已檢核」狀態，不可重複觸發

---

## 五、送單前檢核狀態檢查

> 送單（送簽）時，系統應檢查所有子單是否都已通過下單檢查，未通過則攔截並提示。

**單子單情境：**
- [ ] **5.1** 未點擊「下單檢查」就送單 → 系統攔截，提示「請先完成下單檢查」
- [ ] **5.2** 下單檢查結果為失敗就送單 → 系統攔截，提示「下單檢查未通過」
- [ ] **5.3** 下單檢查全部通過後送單 → 送單流程正常執行

**多子單情境：**
- [ ] **5.4** 其中一個子單未執行下單檢查 → 系統攔截，提示是哪個子單尚未完成檢查
- [ ] **5.5** 其中一個子單下單檢查未通過 → 系統攔截，提示是哪個子單未通過
- [ ] **5.6** 部分子單未執行 + 部分未通過同時存在 → 系統攔截，一次列出所有有問題的子單
- [ ] **5.7** 全部子單都通過下單檢查 → 送單流程正常執行
