# TC-CMP-4308：訂單主動建立料號（前端）測試計劃

| 項目 | 內容 |
|------|------|
| **Jira Issue** | CMP-4308 |
| **功能摘要** | 訂單/訂變單 PM 審核後，送採購審核時自動檢查 T100 料件編號是否存在，若不存在則彈出料件新增視窗 |
| **狀態** | UAT |
| **建立日期** | 2026-04-14 |
| **修訂者** | Raelynn |
| **測試環境** | https://cmp-uat-100.metaage.com.tw/ |
| **測試帳號** | PM 角色 |

---

## 一、功能流程概述

```
PM 點擊「流程送簽」
  → 後端檢查 T100 料件主檔
  → 料件已存在 → 直接送採購審核
  → 料件不存在 → 回傳 erpNumber_not_exist 錯誤碼 + 待建料號清單
    → 前端彈出確認 Dialog「是否要建立料件編號？」
    → 使用者確認 → 開啟「料件新增視窗」（erp-check-modal）
    → PM 填寫必填欄位 → 點擊「確認建立」
    → 顯示「建立中」Dialog（不可關閉）
    → API 回傳結果
      → 成功：顯示成功訊息，停留在當前狀態
      → 失敗：顯示 T100 錯誤訊息，停留在 PM 審核
```

## 三、料件新增視窗 — 表單欄位定義

| # | 欄位 | Key | 類型 | 必填 | Cisco 專用 | 備註 |
|---|------|-----|------|------|-----------|------|
| 1 | 料件編號 | erpNumber | 顯示 | — | ❌ | 系統帶入，不可修改 |
| 2 | 品名 | commodityShortName | input | ✅ | ❌ | |
| 3 | 規格 | specification | input | ✅ | ❌ | |
| 4 | 產品分類碼 | productCategory | select | ✅ | ❌ | 下拉選項由 API 載入 |
| 5 | 料件類別 | materialType | select | ✅ | ❌ | 拉選項由 API 載入 |
| 6 | 產品型式 | productType | select | ✅ | ❌ | 拉選項由 API 載入 |
| 7 | 序號控管 | serialControl | select | ❌ | ❌ | 可不填 |
| 8 | Cisco 分類 | ciscoCategory | select | ✅ | ✅ | 拉選項由 API 載入，僅 Cisco 品牌顯示 |

### 🔹 TC-4308-001：料件已存在 — 直接送採購審核

| 項目 | 內容 |
|------|------|
| **前置條件** | 訂單狀態為「PM審核完畢」，所有料件編號已存在 T100 |
| **角色** | PM |
| **測試類型** | 正向測試 |

**步驟：**
1. 進入 CMP UAT，以 PM 角色登入
2. 進入訂單列表，找到狀態為「PM審核完畢」且料件已存在的訂單

4. 點擊「流程送簽」按鈕

**預期結果：**
- 不彈出料件新增視窗
- 訂單狀態變更為「採購審核」
- 顯示送審成功訊息
